# エッジノードシナリオ

エッジノードシナリオはフィクションではない。ケーススタディでもない。**状況パターン**であり、FF、PP、GG、AAが実世界の制約下でどのように現れるかを示す。

各シナリオは厳密な構造に従う:

1. **状況** — フィールドの条件
2. **制約** — 最も厳しい制限
3. **エッジでの行動 (AA)** — ローカルな意思決定
4. **創発 (GG)** — 計画されていなかった出現
5. **プラットフォームとの相互作用 (PP)** — オプションの同期
6. **フィールドの結果 (FF)** — 維持された自律性

これらのシナリオは**再利用可能なテンプレート**である。コアフレームワークを変更せずに新しいシナリオを追加できる。
