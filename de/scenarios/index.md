# Edge Node Szenarien

Edge Node Szenarien sind keine Fiktion. Sie sind keine Fallstudien. Sie sind **Situationsmuster**, die zeigen, wie FF, PP, GG, AA unter realen Einschränkungen erscheinen.

Jedes Szenario folgt einer strengen Struktur:

1. **Situation** — Feldbedingungen
2. **Einschränkung** — Die schärfste Begrenzung
3. **Handeln am Rand (AA)** — Lokale Entscheidungsfindung
4. **Emergenz (GG)** — Ungeplantes Entstehen
5. **Plattforminteraktion (PP)** — Optionale Synchronisation
6. **Feldergebnis (FF)** — Bewahrte Autonomie

Diese Szenarien sind **wiederverwendbare Vorlagen**. Neue Szenarien können ohne Änderung des Kern-Frameworks hinzugefügt werden.
