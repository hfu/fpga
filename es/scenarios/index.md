# Escenarios de Edge Node

Los escenarios de edge node no son ficción. No son estudios de caso. Son **patrones de situación** que muestran cómo FF, PP, GG, AA aparecen bajo restricciones del mundo real.

Cada escenario sigue una estructura estricta:

1. **Situación** — Condiciones del campo
2. **Restricción** — La limitación más severa
3. **Acción en el Borde (AA)** — Toma de decisiones local
4. **Emergencia (GG)** — Surgimiento no planificado
5. **Interacción con Plataforma (PP)** — Sincronización opcional
6. **Resultado del Campo (FF)** — Autonomía preservada

Estos escenarios son **plantillas reutilizables**. Se pueden agregar nuevos escenarios sin modificar el marco central.
