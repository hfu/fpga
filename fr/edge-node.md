# Edge Node

---

## Définition Conceptuelle

Un edge node est une position dans un système où des décisions sont prises sous contraintes.

Pas un héros. Pas une plateforme. Pas un État. Un lieu spécifique où convergent autonomie, responsabilité et conséquences.

---

## Caractéristiques

Les edge nodes opèrent là où :

- **La connectivité est intermittente** — Le réseau est parfois disponible, pas toujours
- **La coordination centrale n'est pas disponible** — Les décisions ne peuvent pas attendre l'approbation
- **La prise de décision locale est requise** — L'action se produit à la périphérie, pas au centre

Les edge nodes n'échouent pas lors de la déconnexion. La déconnexion est la condition opérationnelle normale.

---

## Scénario

Un ingénieur de terrain dépanne une infrastructure électrique dans un lieu isolé.

**Contexte :**
- Pas de signal de téléphone mobile
- Puissance de batterie limitée à 6 heures
- Les données de diagnostic sont stockées localement
- Les décisions de réparation doivent être prises maintenant

**Ce qui se passe :**

L'ingénieur utilise une tablette avec des outils de diagnostic hors ligne. Le système comprend :
- Manuels d'équipement
- Données historiques de pannes
- Arbres de décision pour problèmes courants
- Stockage de données local

L'ingénieur identifie la panne, effectue la réparation et enregistre le résultat. Quand la connectivité revient, les données se synchronisent avec la plateforme centrale. La plateforme agrège des modèles sur des milliers de réparations. Mais la plateforme n'a pas pris la décision. L'edge node l'a fait.

**Pourquoi c'est important :**

Si le système nécessite un accès cloud pour fonctionner, la réparation ne se produit pas. L'edge node est bloqué par l'architecture, pas par la capacité. C'est un échec de conception, pas un échec de terrain.

---

## Le Changement

Ancien cadre : État contre individu.

Nouveau cadre : Plateforme et terrain.

Le terrain a toujours existé. Cliniques isolées. Travail offshore. Réponse aux catastrophes. Stations de recherche. Le changement n'est pas que le terrain devienne réel. C'est de concevoir des systèmes qui reconnaissent la réalité du terrain comme point de départ, pas comme cas limite.
