# 邊緣節點情境

邊緣節點情境不是虛構。不是案例研究。它們是**情境模式**，展示 FF、PP、GG、AA 如何在真實世界限制下顯現。

每個情境遵循嚴格結構：

1. **情境** — 場域條件
2. **限制** — 最嚴格的限制
3. **邊緣行動 (AA)** — 本地決策
4. **湧現 (GG)** — 非計劃的出現
5. **平台互動 (PP)** — 可選同步
6. **場域結果 (FF)** — 保留的自主權

這些情境是**可重複使用的模板**。可以在不修改核心框架的情況下添加新情境。
